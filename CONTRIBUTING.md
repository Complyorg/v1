# Contributing to the Comply.org Attestation Standard

Thank you for your interest in improving the standard. Contributions are welcome
from privacy engineers, lawyers, auditors, platform developers, and anyone
working in data protection.

## How to Contribute

1. **Open an issue** to discuss proposed changes before submitting a pull
   request. This is especially important for changes to the schema or core
   principles.
2. **Fork the repository** and create a feature branch.
3. **Submit a pull request** with a clear description of the change and its
   rationale.

## What We Accept

- **Schema changes**: New optional fields, improved descriptions, additional
  validation constraints.
- **Specification clarifications**: Improved wording, additional examples,
  edge-case guidance.
- **Templates and generators**: Reference implementations in any language.
- **Examples**: Real-world or realistic attestation documents.
- **Translations**: The specification in other languages.

## What Requires an RFC

Changes to **required fields**, **core principles**, or **assessment statuses**
are breaking changes that affect all implementations. These require a formal
proposal (open an issue tagged `rfc`) and community discussion before merging.

## Code of Conduct

Be respectful, constructive, and professional. This is a standards project
serving the global privacy community.
